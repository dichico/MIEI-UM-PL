# Definição da GIC que permitirá criar a linguagem em si
# Regras de Derivação que dizem como combinar os símbolos terminais

& é usado para representar o vazio e NÃO É UM SÍMBOLO TERMINAL porque não tem materialização
Nesta caso não vamos usar esse símbolo

FicheiroPug         --> DECLINICIAL HEAD ConteudoHead BODY ConteudoBody

ConteudoHead        --> Titulo
                     |  Titulo AtributoHandler 

Titulo              --> string

AtributoHandler     --> '(' Atributos ')'

Atributos           --> Atributos ',' Atributo
                     |  Atributo

Atributo            --> string

ConteudoBody        --> TagHandler ????

TagHandler          --> Tags string

Tags                --> 'h1' 
                     |  'h2'
                     |  'h3'
                     |  'h4'
                     |  'h5'
                     |  'h6'
                     |  'p'

DivHandler          --> '#' string '.' string
                     |  '.' string  
                     |  '#' string

# De acordo com esta gramática, podemos concluir:

1. Existem 2 produções diferentes
2. Existem 1 símbolos não-terminais ->  ConteudoHead ConteudoBody 
3. Existem 4 símbolos terminais     ->  DECLINICIAL, HEAD, BODY
